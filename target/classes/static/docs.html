Getting Started

CursorHeat is a powerful analytics tool that helps you understand user behavior through mouse tracking and heatmaps. This guide will help you get started with integrating CursorHeat into your application.

Installation

Add the CursorHeat script to your HTML: 

<script src="https://cdn.cursorheat.com/tracker.js" data-project-id="YOUR_PROJECT_ID"></script>

API Reference

Authentication

All API requests require authentication using your API key. Include it in the Authorization header: 

Authorization: Bearer YOUR_API_KEY

Endpoints

Events

POST /api/v1/events
GET /api/v1/events
GET /api/v1/events/{id}

Sessions

GET /api/v1/sessions
GET /api/v1/sessions/{id}
GET /api/v1/sessions/{id}/events

Heatmaps

GET /api/v1/heatmaps
GET /api/v1/heatmaps/{id}
POST /api/v1/heatmaps

Events

CursorHeat tracks various types of events:

  • Mouse movements
  • Clicks
  • Scrolls
  • Form interactions
  • Page views

Example event payload: 

{
  "type": "click",
  "x": 100,
  "y": 200,
  "timestamp": "2024-03-20T12:00:00Z",
  "page": "/home",
  "sessionId": "session_123"
}

Heatmaps

Heatmaps visualize user interaction data on your pages. You can create heatmaps for:

  • Mouse movements
  • Clicks
  • Scroll depth

Example heatmap configuration: 

{
  "type": "click",
  "page": "/home",
  "startDate": "2024-03-01",
  "endDate": "2024-03-20",
  "threshold": 0.5
}

Sessions

Sessions represent user visits to your website. Each session contains:

  • Start and end times
  • Page views
  • Events
  • User information

Integrations

CursorHeat integrates with popular tools and services:

  • Google Analytics
  • Slack
  • Custom webhooks

Example webhook configuration: 

{
  "url": "https://your-webhook.com/events",
  "events": ["click", "move", "scroll"],
  "secret": "your-webhook-secret"
}
